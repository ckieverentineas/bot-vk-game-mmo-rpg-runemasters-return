# Quickstart

## 1. Установить зависимости

```bash
npm install
```

## 2. Сгенерировать Prisma Client

```bash
npm run db:generate
```

## 3. Поднять схему базы

Для новой базы:

```bash
npm run db:push
```

Если старая база мешает и её можно пересоздать:

```bash
npm run db:push -- --accept-data-loss
```

## 4. Засеять мир

```bash
npm run db:seed
```

Сид готовит стартовую базу новой архитектуры: биомы, шаблоны мобов и базовые world-данные.

## 5. Настроить `.env`

Минимальный набор:

```env
VK_TOKEN=your_vk_group_token_here
# Optional. Можно оставить пустым, бот сам определит id группы.
VK_GROUP_ID=
DATABASE_URL="file:./prisma/dev.db"
```

## 6. Проверить проект перед запуском

```bash
npm run check
```

Эта команда прогоняет:

- `typecheck`
- `test`
- `build`

## 7. Проверить релизный статус

```bash
npm run release:status
```

Скрипт показывает текущую пользовательскую версию по правилу `100` коммитов = `1.00`.

## 8. Запустить бота

Для локальной разработки:

```bash
npm run dev
```

Для production-сборки:

```bash
npm run build
npm start
```

## Базовый сценарий игры

1. написать `начать`;
2. пройти учебный бой в нулевом биоме или нажать `пропустить обучение`;
3. открыть `профиль` и распределить очки;
4. начать бой через `исследовать`;
5. провести ход через `атака`;
6. при желании вернуться в `локация` и снова открыть обучение;
7. проверить `руна` и `алтарь`.

## Что важно знать

- transport-слой команд централизован в `src/vk/commands/catalog.ts`;
- клавиатуры собираются через общий builder из `src/vk/keyboards/index.ts`;
- сериализация боевого состояния централизована в `src/shared/utils/json.ts`;
- если активный бой в базе завис в неконсистентном состоянии, он будет автоматически завершён при следующем входе в бой;
- персонажа теперь можно удалить из интерфейса через кнопку/команду `удалить персонажа`;
- после изменений в `prisma/schema.prisma` нужно заново запускать `npm run db:generate`;
- изменения в пользовательском поведении должны попадать в `CHANGELOG.md`.
