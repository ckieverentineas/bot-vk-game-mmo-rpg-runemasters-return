# Economy Loop v1 Design

## Статус

- Статус: `approved for planning`
- Дата: `2026-04-28`
- Цель: довести Runemasters Return до полноценного замкнутого игрового цикла, где каждый ресурс, валюта, предмет и награда имеют понятный источник, применение и следующий смысловой шаг.

## Проблема

Текущая игра уже имеет рабочие контуры боя, исследования, рун, трофеев, квестов, бестиария, пилюль и мастерской. Но часть экономики всё ещё ощущается как набор ранних срезов:

- материалы имеют несколько применений, но не все одинаково важны;
- `✨ Сияние` уже добывается, но не должно стать бытовой валютой для рутины;
- `пыль` используется в рунах, но может стать шире как универсальная денежная валюта;
- чертежи сейчас хранятся как `blueprintCode + quantity`, поэтому не могут быть редкими, секретными, квестовыми или уникальными;
- мастерская показывает каталог шаблонов, а не только реальные найденные возможности игрока;
- качество созданного предмета не зависит от параметров выпавшего чертежа и навыков игрока.

Игрок должен чувствовать не ранний доступ, а завершённую RPG-петлю: исследование даёт добычу, добыча открывает выбор, выбор меняет билд, билд открывает более сложные цели, а лишние ресурсы не превращаются в мёртвый склад.

## Цели

- Составить единую карту `источник -> применение -> цель -> следующий шаг` для всех ресурсов.
- Закрыть сиротские ресурсы без превращения игры в бесконечную разработку.
- Переработать чертежи в уникальные одноразовые crafting opportunities.
- Сделать мастерскую экраном реальных возможностей игрока, а не справочником всех шаблонов.
- Дать `✨ Сиянию` роль редкого ресурса для уникальных плюшек и особенностей, а не рутины.
- Расширить роль пыли как универсальной валюты для рун, покупок и подготовленного после Economy Loop v1 обмена.
- Сохранить replay-safety, exact-once награды и command intent rails.

## Вне рамок

- Полноценный рынок между игроками.
- Auction house.
- Pay-for-power.
- Сезонная экономика.
- Массовый ребаланс всех врагов и локаций.
- Глубокая процедурная система аффиксов на десятки свойств.
- Ломка текущих рунных школ и боевой системы.

Эти направления не входят в Economy Loop v1 и требуют отдельного product review после стабилизации базовой экономики.

## Подход

Выбран подход `Economy Loop v1`: закрыть экономику на текущих системах, не добавляя крупный новый жанровый слой.

Главная петля:

```text
Исследование -> встреча или событие -> бой / выбор -> трофеи и награды -> руны, мастерская, пилюли, чертежи -> усиление или новый стиль -> более опасное исследование
```

Вторичные петли:

- `школы -> печати -> мастерство -> особые испытания`;
- `трофеи -> материалы -> чертёж / предмет / пилюля -> новый бой`;
- `бестиарий -> знание врагов -> milestone-награды -> редкие чертежи`;
- `daily trace -> мягкий бонус -> редкое Сияние -> уникальные особенности`.

## Роли ресурсов

| Ресурс | Источники | Применение | Роль в цикле |
| --- | --- | --- | --- |
| Пыль | бой, квесты, бестиарий, daily trace, продажи NPC после V1 | руны, перековка, покупки, подготовка к обмену | универсальная денежная валюта |
| Сияние | daily trace, редкие события, важные квесты, milestone-награды | уникальные плюшки, пробуждение особенностей, редкие мастерские решения | редкая валюта исключительных решений |
| Осколки рун | бой, школа, квесты, редкие награды | создание рун по редкости | рунная прогрессия |
| Кожа | звери, трофеи, события | пилюли, лёгкое снаряжение, инструменты | ранний физический материал |
| Кость | звери, нежить, трофеи | пилюли, оружие, инструменты | ранний боевой материал |
| Трава | события, природные враги, daily trace | пилюли, алхимия, мягкое восстановление | ресурс поддержки |
| Эссенция | магические враги, редкие трофеи | руны UNUSUAL+, фокус, особые крафты | магический материал |
| Металл | бронированные враги, шахтные/руинные сцены | оружие, броня, руны EPIC+ | прочный материальный прогресс |
| Кристалл | редкие враги, milestone, поздние сцены | ремонт UL, редкие руны, особые чертежи | редкий late-game материал |
| Пилюли | мастерская | бой и восстановление между встречами | расходный safety valve |
| Чертежи | квесты, трофеи, бестиарий, события | одноразовый крафт предмета или ремонтного инструмента | превращают добычу в выбор |
| Снаряжение | крафт по уникальному чертежу | экипировка, узкие бонусы, прочность | build-слой рядом с рунами |

## Пыль

Пыль становится универсальной денежной валютой.

Сохраняются текущие применения:

- создание рун;
- перековка рун.

Добавляются применения Economy Loop v1:

- покупка базовых расходников у мастерской или торговца;
- покупка простых материалов в ограниченном количестве;
- оплата части работы при крафте сложных предметов.

Подготовка к обмену остаётся только архитектурным запасом: Economy Loop v1 не вводит рынок игроков.

Пыль не должна заменять все материалы. Если предмет физический, материалы остаются главным ограничителем, а пыль оплачивает работу, обработку или торговую цену.

## Сияние

Сияние не используется для рутины.

Запрещённые применения:

- массовый крафт обычных пилюль;
- базовая перековка каждой руны;
- постоянная покупка обычных материалов;
- прямой pay-to-win рост статов.

Разрешённые применения:

- пробудить особую черту уже найденного редкого чертежа;
- пробудить особую черту квестового или школьного чертежа;
- закрепить одну уникальную особенность результата в безопасном диапазоне;
- открыть особую мастерскую возможность;
- активировать редкий ритуал качества;
- получить косметический или narrative-facing знак прогресса;
- подсветить уникальную цель после важного рубежа.

Сияние должно ощущаться как редкий ключ к особенным решениям. Игрок не обязан тратить его каждый день, но рад видеть, что оно накапливается ради заметных моментов.

## Уникальные чертежи

Чертёж становится уникальным экземпляром, а не количеством шаблона.

Новая модель игрока:

```text
PlayerBlueprintInstance
- id
- playerId
- blueprintCode
- rarity
- sourceType
- sourceId
- discoveryKind
- quality
- craftPotential
- modifierSnapshot
- status
- createdAt
- discoveredAt
- consumedAt
```

Значения:

- `id` — конкретный экземпляр чертежа.
- `blueprintCode` — семейство рецепта: тесак, куртка, инструмент, редкий амулет.
- `rarity` — редкость выпавшего чертежа.
- `sourceType` — `TROPHY`, `QUEST`, `BESTIARY`, `DAILY_TRACE`, `EVENT`, `SCHOOL_TRIAL`.
- `sourceId` — враг, квест, milestone или событие.
- `discoveryKind` — `COMMON`, `SECRET`, `QUEST`, `SCHOOL`, `REPAIR`.
- `quality` — базовое качество экземпляра.
- `craftPotential` — диапазон или профиль результата.
- `modifierSnapshot` — зафиксированные свойства чертежа.
- `status` — `AVAILABLE`, `CONSUMED`, `EXPIRED`.

`quantity` уходит из player-facing модели. Если игрок нашёл три похожих чертежа, он видит три экземпляра, отличающиеся редкостью, качеством или источником.

## Типы чертежей

| Тип | Как появляется | Как показывается | Что даёт |
| --- | --- | --- | --- |
| Обычный | бой, трофеи, события | сразу открыт | понятный крафт |
| Секретный | скрытые условия, редкие трофеи, бестиарий, события | появляется только после открытия | даёт редкую или необычную возможность |
| Квестовый | главы пути, школы, важные NPC-сцены | виден как цель или награда | создаёт особый предмет |
| Школьный | испытания школы, печати, mastery | связан со школой игрока | усиливает стиль, не ломая баланс |
| Ремонтный | milestone, редкие события, мастерская | отдельный инструмент | чинит подходящий UL-предмет |

Секретный чертёж не продаётся и не раскрывается за Сияние. До выполнения условия он вообще не должен выглядеть как обычный пункт мастерской. Игрок может получить намёк через мир, квест, бестиарий или странное трофейное событие, но конкретный экземпляр появляется только после открытия условия.

После открытия он показывается как найденная редкость:

```text
📜 Секретный чертёж: Набор свежевателя
Открыт после внимательной работы с трофеями зверей.
```

Сияние может применяться после этого как отдельное редкое решение: пробудить особую черту чертежа или закрепить один безопасный уникальный нюанс результата. Оно не является способом получить сам секретный чертёж.

## Качество результата

Качество созданного предмета зависит от трёх вещей:

1. параметров чертежа;
2. навыков игрока;
3. материалов и выбранного способа доводки.

Чертёж задаёт базовый потенциал:

- редкость;
- базовую прочность;
- допустимые бонусы;
- возможный quality floor;
- возможную особенность.

Навык игрока улучшает результат мягко:

- повышает шанс лучшего качества;
- снижает шанс слабого броска;
- добавляет небольшую прочность;
- иногда открывает дополнительную строку в пределах чертежа;
- улучшает экономию материалов только в явно ограниченных случаях.

Навык не должен превращать плохой чертёж в лучший предмет мира. Хороший чертёж остаётся важной добычей.

## Качество и редкость

Редкость отвечает за потенциал, качество отвечает за исполнение.

Пример:

| Редкость чертежа | Возможный предмет | Что меняется |
| --- | --- | --- |
| COMMON | простой инструмент | низкая прочность, одна польза |
| UNCOMMON | устойчивый инструмент | выше прочность или узкий бонус |
| RARE | особый инструмент | сильная специализация |
| EPIC | мастерская вещь | заметная особенность |
| QUEST | сюжетный предмет | уникальная роль без обязательной силы |

Качество внутри редкости:

- `ROUGH` / `Грубое` — рабочее, но слабое исполнение;
- `STURDY` / `Крепкое` — нормальный результат;
- `FINE` / `Тонкое` — хороший результат;
- `MASTERWORK` / `Мастерское` — редкий лучший результат.

В домене хранится код качества, а player-facing слой показывает русское название.

## Мастерская

Мастерская должна показывать только реальные действия игрока.

Убрать player-facing показ всех шаблонов каталога. Экран строится от:

- найденных доступных чертежей;
- секретных чертежей, которые игрок уже открыл;
- квестовых чертежей, которые игрок уже получил или почти открыл;
- созданных предметов;
- доступных ремонтов;
- пилюль и расходников, которые можно создать или применить.

Если у игрока нет чертежей, экран не показывает весь каталог как закрытый список. Он показывает короткую цель:

```text
📜 Чертежей пока нет.
Ищите их в трофеях, Книге путей и Бестиарии.
```

Если есть секретный чертёж:

```text
📜 Секретный чертёж: Набор свежевателя
Источник: трофеи зверей.
Можно создать инструмент или пробудить особую черту за Сияние.
```

Если есть открытый чертёж:

```text
📜 Тесак следопыта
Редкость: RARE · качество: FINE · источник: Бестиарий
Создаст оружие с прочностью 16-20 и звериным трофейным бонусом.
```

## Снаряжение

Снаряжение остаётся отдельным от рун.

Руны дают:

- школьную идентичность;
- активные способности;
- магические пассивы;
- главный build-ритм.

Снаряжение даёт:

- узкие физические бонусы;
- инструменты добычи;
- прочность;
- situational utility;
- связь с материалами и чертежами.

Предмет не должен быть просто второй руной без магии. Хороший предмет отвечает на вопрос: “что я теперь могу делать иначе?”

## Источники чертежей

Начальные источники:

- квест `awakening_empty_master` даёт первый простой инструментальный чертёж;
- бестиарий за первый набор встреч в локации даёт тематический чертёж;
- kill milestone даёт редкий или секретный чертёж;
- трофейные действия могут дать чертёж, связанный с типом врага;
- школьный путь может дать квестовый чертёж, связанный со стилем школы.

Распределение должно быть ограниченным. Чертежи не должны сыпаться в каждом бою, иначе уникальность исчезнет.

Первый вертикальный срез секретного чертежа: `Набор свежевателя`.

- Источник: ранний bestiary или kill milestone по звериному врагу.
- Условие открытия: несколько успешных трофейных действий по зверям или соответствующий bestiary milestone.
- Начальное состояние до открытия: экземпляра у игрока нет, в мастерской он не показывается.
- Состояние после открытия: `discoveryKind = SECRET`, `status = AVAILABLE`.
- Результат: уникальный экземпляр чертежа `skinning_kit`.
- Польза: предмет-инструмент усиливает трофейные действия по зверям и тратит прочность на применении.
- Опциональное применение Сияния: пробудить одну особую черту инструмента после открытия чертежа.

## Применение материалов

Каждый материал получает минимум два значимых применения:

- `кожа`: лёгкая броня, инструменты, пилюля восстановления;
- `кость`: оружие, инструменты, пилюли;
- `трава`: пилюли, природные чертежи, поддержка;
- `эссенция`: руны UNUSUAL+, фокус, магические предметы;
- `металл`: оружие, броня, руны EPIC+;
- `кристалл`: ремонт UL, редкие чертежи, поздние рунные действия.

Если материал начинает копиться без применения, Economy Loop evidence должен помечать это как gap.

## Навыки

Навыки становятся экономическими модификаторами, но не прямой кнопкой силы.

Примеры:

- `alchemy` повышает выход пилюль или качество расходника;
- `skinning` улучшает материалы зверей и шанс соответствующих чертежей;
- `reagent_gathering` улучшает траву, эссенцию и природные находки;
- `essence_extraction` помогает магическим материалам и секретным чертежам;
- новый `crafting.workshop` влияет на качество снаряжения, доводку мастерских чертежей и ремонт.

Навыки должны быть проверяемыми чистыми функциями: входом являются чертёж, материалы и skill snapshot; выходом является deterministic craft preview или результат с server-authoritative random.

`crafting.alchemy` остаётся навыком пилюль. Economy Loop v1 добавляет отдельный `crafting.workshop`, чтобы качество оружия, брони и инструментов не зависело от алхимии.

## Крафт

Крафт принимает конкретный `blueprintInstanceId`.

Шаги:

1. загрузить игрока, инвентарь, навыки и экземпляр чертежа;
2. проверить, что чертёж принадлежит игроку и доступен;
3. проверить, что чертёж доступен и не является будущей квестовой целью без экземпляра;
4. рассчитать preview результата;
5. проверить материалы и пыль;
6. выполнить command intent guarded mutation;
7. списать материалы, пыль и конкретный чертёж;
8. создать предмет с quality snapshot;
9. записать economy transaction telemetry.

Повтор команды не должен создавать второй предмет и не должен повторно списывать ресурсы.

## Секретные чертежи

Секретный чертёж — это не платная расшифровка. Это редкая находка, которую игрок получает за скрытое условие, внимательное трофейное действие, milestone или необычную сцену.

Шаги:

1. игра проверяет секретное условие в трофее, бестиарии, квесте или событии;
2. при выполнении условия создаётся `PlayerBlueprintInstance` с `discoveryKind = SECRET`;
3. мастерская показывает уже найденный секретный чертёж;
4. игрок решает, создать предмет сейчас или оставить чертёж на потом;
5. крафт тратит конкретный экземпляр exact-once.

Секретный чертёж должен ощущаться как “я это нашёл”, а не “я это купил”.

## Пробуждение особенности за Сияние

Сияние может применяться к уже найденному редкому, секретному, квестовому или школьному чертежу.

Шаги:

1. игрок уже владеет доступным экземпляром чертежа;
2. чертёж поддерживает `radianceFeature`;
3. мастерская показывает, какую особенность можно пробудить;
4. игрок тратит Сияние;
5. чертёж получает обновлённый `modifierSnapshot`;
6. будущий крафт создаёт предмет с пробуждённой особенностью.

Это не должно быть обязательной рутиной для каждого предмета. У большинства обычных чертежей нет `radianceFeature`.

## Покупки и торговля

В Economy Loop v1 пыль получает NPC-покупки без полноценного рынка.

Первый контур покупок живёт внутри `🛠 Мастерской` как раздел `Лавка мастерской`, а не как отдельный экран. Это снижает навигационный шум и сразу связывает покупку с применением.

Безопасные покупки:

- малые материалы с ограничением по ассортименту;
- базовые пилюли;
- дешёвые COMMON-чертежи низкого потенциала;
- услуга восстановления или простая мастерская работа.

Нельзя продавать за пыль:

- лучшие секретные чертежи;
- квестовые чертежи;
- school-defining предметы;
- прямую победу над progression gate.

Это подготавливает модель к обмену после V1, но не вводит рынок игроков в текущем срезе.

## Данные и миграция

Нужна новая persistence-модель для экземпляров чертежей.

Минимальная новая таблица:

```text
PlayerBlueprintInstance
- id String @id
- playerId Int
- blueprintCode String
- rarity String
- sourceType String
- sourceId String?
- discoveryKind String
- quality String
- craftPotential String
- modifierSnapshot String
- status String
- createdAt DateTime
- discoveredAt DateTime?
- consumedAt DateTime?
```

Старую таблицу `PlayerBlueprint` нельзя резко удалять в первом шаге. Нужен compatibility bridge:

- старые записи читаются как legacy stacks;
- миграция создаёт экземпляры по quantity;
- каждый legacy-экземпляр получает безопасные значения `rarity`, `quality`, `sourceType = LEGACY`;
- после миграции мастерская работает с экземплярами;
- legacy API сохраняется временно только там, где нужно для старых reward payloads.

## Контракты наград

`ResourceReward.blueprintDelta` больше не должен быть единственным будущим способом выдачи чертежей.

Новый контракт должен поддерживать:

```text
blueprintDrops: readonly BlueprintDropIntent[]
```

Где `BlueprintDropIntent` описывает:

- `blueprintCode`;
- `rarity`;
- `discoveryKind`;
- `sourceType`;
- `sourceId`;
- `qualityHint`;
- `modifierProfileCode`.

Для совместимости `blueprintDelta` временно преобразуется в обычные экземпляры `discoveryKind = COMMON`.

## UX-правила

- Не показывать игроку весь каталог шаблонов как закрытый список.
- Сначала показывать готовые действия.
- Потом показывать найденные, но не готовые действия.
- Секретные чертежи показывать только после открытия, как находки с источником и короткой причиной открытия.
- Для каждого ресурса в инвентаре показывать ближайшее применение, если оно известно.
- Если применение ещё не открыто, говорить через мир: “мастерская пока не знает, как это применить”, но не создавать ощущение сломанной системы.
- Ошибки нехватки ресурсов показывать человечески, без `JSON.stringify`.

## Критерии готовности

- У каждого ресурса из `InventoryView`, `gold` и `radiance` есть documented source и sink.
- Мастерская не показывает craft-шаблоны, если у игрока нет соответствующего чертежа.
- Найденные чертежи являются уникальными экземплярами.
- Секретный чертёж появляется только после выполнения условия и не покупается за Сияние.
- Сияние может пробудить особую черту уже найденного подходящего чертежа.
- Сияние не используется для рутинного массового крафта.
- Пыль используется как универсальная валюта минимум в рунах и одном безопасном purchase/workshop контуре.
- Качество созданного предмета зависит от чертежа и навыков игрока.
- Крафт конкретного чертежа exact-once и replay-safe.
- Старые сохранения с `PlayerBlueprint` не ломаются.
- `npm run check` проходит.
