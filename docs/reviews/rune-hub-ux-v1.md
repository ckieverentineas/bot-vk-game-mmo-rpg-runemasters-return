# Rune Hub UX v1

> Актуальный UX baseline для рун после отмены player-facing support-slot модели.

## Purpose

Сделать rune hub быстрым, читаемым и пригодным для роста сборки без свалки из всех действий на одном экране.

## Locked decisions

- Первый экран показывает список до **5 рун на странице**.
- В списке рядом с руной видно, надета ли она и в каком слоте.
- Кнопки `1..5` выбирают руну из текущей страницы.
- Навигация страницы идёт отдельными кнопками `назад / вперёд`, без one-by-one browsing.
- После выбора руны открывается отдельная карточка: статы, пассивные эффекты, активные навыки, стоимость маны, откат и действия.
- `Надеть` автоматически выбирает первый свободный слот; если свободных слотов нет, обычный путь заменяет первый слот, а целевые slot-команды остаются только compatibility-поверхностью.
- `Снять` и `Распылить` показываются только в контексте выбранной руны.
- Два стартовых слота равноправны: обе руны дают полные статы, пассивы и активные действия.

## Why this slice

- игрок видит коллекцию как список, а не как один огромный технический дамп;
- выбор руны и действие с ней разделены, поэтому экран меньше шумит;
- статус `надета` читается до входа в карточку;
- карточка объясняет, зачем руна нужна в бою;
- текущая модель не обещает support-slot и не мешает будущим 3+ слотам через progression tree.

## Explicitly out of scope

- player-facing support-slot;
- passive-only second slot;
- drag/drop, compare matrix и сложные presets;
- slot swap mid-battle;
- cross-slot proc-web как обязательная ранняя механика;
- direct page number jump, пока коллекция не стала достаточно большой, чтобы это реально ускоряло UX.
