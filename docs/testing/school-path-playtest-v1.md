# Ручной playtest / evidence pass — Пламя и Твердь

## Цель

Проверить ранний school-first vertical slice до первой полноценно читаемой цепочки:

`novice elite -> first unusual sign -> equip sign -> school miniboss -> first rare seal`

Сценарий проходится отдельно для:

- Пламени
- Тверди
- Бури
- Прорицания

## Что считаем успехом

Игрок:

- понимает, что идёт именно по пути своей школы, а не по generic PvE loop;
- получает понятный следующий шаг после каждого milestone;
- не теряется между наградой и её реальной установкой в сборку;
- воспринимает `unusual sign` и `rare seal` как разные school-вехи, а не просто как новые предметы;
- хочет сделать ещё один бой после miniboss, а не выйти в menu wandering.

---

## Формат evidence

Для каждого шага сохранить:

- 1 скрин экрана/сообщения;
- короткую заметку в формате `понятно / непонятно / перегружено / хочется нажать дальше`;
- если ожидание игрока и реальный результат расходятся — 1 фраза где именно.

После ручного прогона дополнительно запустить:

- `npm run release:school-evidence`
- `npm run release:evidence` (по умолчанию за последние 7 дней; при необходимости уточнить окно через `--since` / `--until`; date-only значения считаются как UTC-границы дня)

И приложить оба markdown-отчёта как часть evidence pass: узкий school-path funnel и общий release evidence с return/next-goal/QA контекстом.

## Сценарий A — Пламя

1. Дойти до novice elite школы Пламени.
2. Победить `Пепельную ведунью`.
3. Получить первую `UNUSUAL` руну школы.
4. Понять из текста, что это не просто дроп, а `испытание школы пройдено`.
5. Надеть первый знак школы.
6. Дойти до `Пепельной матроны`.
7. Победить school-aligned miniboss.
8. Получить первую `RARE` печать школы.
9. Понять, что это уже не первый знак, а следующий ранг school payoff.

- Понял ли игрок разницу между `знак школы` и `печать школы`?
- Хотелось ли продолжить путь Пламени после получения rare-награды?

## Что проверить по telemetry после прогона

Использовать **реальные текущие события**, а не выдуманные funnel-имена.

### Базовая цепочка novice path

Проверить наличие и порядок:

1. `school_novice_elite_encounter_started`
2. `post_session_next_goal_shown` с `suggested_goal_type = hunt_school_elite`
3. `reward_claim_applied` с:
   - `is_school_novice_aligned = true`
   - `novice_target_reward_rarity = UNUSUAL`
4. `school_novice_follow_up_action_taken` с `action_type = open_runes` или `equip_school_sign`
5. `loadout_changed` для установки first sign

### Continuation к miniboss

Проверить наличие и порядок:

1. `school_novice_follow_up_action_taken` с `action_type = start_next_battle`
2. `post_session_next_goal_shown` с `suggested_goal_type = challenge_school_miniboss`
3. `reward_claim_applied` для rare-награды с:
   - `battle_id`
   - `battle_school_code`
   - `reward_rune_rarity = RARE`
4. при необходимости следующий `school_novice_follow_up_action_taken` / `loadout_changed`, если игрок надел rare seal

### На что смотреть отдельно

- нет дублей `reward_claim_applied` на один и тот же бой;
- `equip_school_sign` не логируется повторно на replay того же intent;
- `open_runes` / `start_next_battle` дают понятную последовательность после novice/miniboss payoff;
- `post_session_next_goal_shown` не расходится с тем, что реально было видно на экране;
- по timestamp/session видно, что игрок не завис между `reward_claim_applied` и следующей осмысленной action.

## Решение после прогона

Если path признан слабым, явно указать одно из трёх:

- `iterate` — path почти работает, но теряет игрока в конкретной точке;
- `delay` — path интересный, но слишком сырой для 1.0;
- `cut` — path не даёт enough value и только раздувает scope.
